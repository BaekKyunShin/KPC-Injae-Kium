# 인재키움 사이트 작업 가이드

KPC 직무교육센터 "인재키움" 정적 사이트입니다.

## 데이터 갱신 워크플로우

과정 일정 변경 / 신규 과정 추가는 **엑셀 → 빌드 스크립트** 한 번이면 됩니다.

```
1. 사내 시스템에서 엑셀 추출 → data/ 폴더에 저장 (파일명 자유)
2. npm run build:courses
3. 새 브랜치 → 커밋 → PR → 머지 (main 직접 푸시 금지)
```

빌드 스크립트(`scripts/build-courses.mjs`)가 자동으로:
- `data/`의 가장 최근 `.xlsx`를 읽어 `courses_grouped.json` 재생성
- `index.html` / `courses.html`의 **정적 카운트 값 갱신** (히어로 숫자, 마퀴, stat 카드, 목록 페이지 카운트)

## 직접 수정 금지 파일

자동 생성/갱신되는 값들은 **수동으로 만지지 말 것**. 다음 빌드에서 덮어써집니다.

- `courses_grouped.json` — 엑셀에서 생성
- `index.html` / `courses.html` 안의 **`<!--C-->213<!--/C-->`** (과정 수) 와 **`<!--S-->607<!--/S-->`** (차수 수) 마커 사이의 숫자
  - 마커 자체(`<!--C-->`, `<!--/C-->`, `<!--S-->`, `<!--/S-->`)는 빌드 스크립트가 식별하는 anchor이므로 **삭제·이동 금지**
  - 새 위치에 카운트 표시를 추가하고 싶으면 마커로 감싸기만 하면 됨: `<b><!--C-->0<!--/C--></b>개 과정`

## 기타 규약

- 엑셀 원본(`data/*.xlsx`, `*.xls`, `*.csv`)은 **공개 레포에 커밋 금지** (.gitignore 처리됨). 내부 매출/강사료/HRD 정보 포함.
- 운영 흐름 상세는 `data/README.md` 참조.
